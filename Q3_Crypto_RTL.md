# 🧩 ענף 3: ארכיטקטורות קריפטוגרפיות ותכנון RTL (שאלה 3 במבחן)

**מהות השאלה:** יינתן שרטוט בלוקים של סיבוב בודד (Round) מתוך אלגוריתם הצפנה לא מוכר. יש לזהות את הארכיטקטורה שלו, להסביר את מטרותיה התיאורטיות, ולתרגם את השרטוט לקוד Verilog קומבינטורי נקי המבוסס על מוקשי חיווט ופקודות `assign`.

---

## 1. זיהוי ארכיטקטורות מתוך שרטוט והבדלים בחומרה
במבחן תישאל "לאיזו משפחה שייך האלגוריתם?", אלו האפשרויות:

### א. רשת פייסטל (Feistel Network)
* **איך מזהים בשרטוט?** המידע הנכנס מפוצל לשני חצאים (לרוב ימין ושמאל). חצי אחד (למשל ימין) חוצה את הקווים ויורד למטה ללא שום שינוי (הופך לשמאל החדש: $L_{new} = R_{old}$). החצי השני עובר פעולת XOR מול פונקציה מורכבת ($F$) שניזונה מהחצי הראשון ומהמפתח ($R_{new} = L_{old} \oplus F(R_{old}, Key)$).
* **תפקיד פונקציה F:** "המנוע" של ההצפנה. מייצרת את הבלבול (Confusion). היתרון העצום שלה הוא שהיא **לא חייבת להיות הפיכה מתמטית**.
* **יתרון חומרתי בולט:** חסכוני מאוד בשטח סיליקון (Area). משתמשים **באותה חומרה בדיוק** להצפנה ולפענוח, רק הופכים את סדר הזנת המפתחות.
* **דוגמאות:** DES, GOST, SEED.

### ב. רשת SPN (Substitution-Permutation Network)
* **איך מזהים בשרטוט?** אין פיצול לחצאים! כל בלוק המידע מועבר במקביל כמקשה אחת דרך שכבות של החלפה (קופסאות S-Box) ולאחר מכן ערבוב ופיזור (Permutation / P-Box).
* **מטרות (לפי שאנון):** ה-S-Boxes מייצרים **Confusion** (בלבול סטטיסטי בין המפתח לטקסט), וה-P-Boxes מייצרים **Diffusion** (פיזור - שינוי ביט אחד בקלט ישפיע על ביטים רבים בפלט).
* **חיסרון חומרתי:** דורש חומרה כפולה. מכיוון שהמידע לא מפוצל ל-XOR כמו בפייסטל, חובה לבנות רכיבי Inverse S-Box ו-Inverse P-Box נפרדים לחלוטין עבור מעגל הפענוח.
* **דוגמאות:** AES (Rijndael).

### ג. רשת ARX (Add, Rotate, XOR) - "המוקש"
* **איך מזהים בשרטוט?** השרטוט לא מפצל מידע (לא Feistel) ואין בו שום קופסאות S-Box להחלפה (לא SPN). יש בו רק קווים שמתחברים בעזרת חיבור מתמטי חרוץ (+), שערי XOR, ופעולות הזזה מעגלית (<<<).
* **תשובה אופיינית למבחן:** "אין שום קשר או דמיון בין הארכיטקטורה הזו ל-Feistel או SP Network. זוהי ארכיטקטורת ARX המסתמכת על פעולות לוגיות ואריתמטיות פשוטות ומהירות".
* **דוגמאות:** LEA.

## 2. המרת השרטוט ל-Verilog (Combinational RTL)
הקוד שתידרש לכתוב יהיה **תמיד** ללא בלוקים של שעון (`always @(posedge clk)`). בשאלות אלו, המרצה מבקש "לוגיקה קומבינטורית בלבד", מה שאומר שיש להשתמש בחוטים (`wire`) ובהשמות רציפות (`assign`) בלבד.

* 💡 **הערה חומרתית (Barrel Shifter):** השימוש ב-`assign` עבור הזזות מבטיח שהסינתסייזר יפרוס על הסיליקון רשת מרבבים מיידית (Barrel Shifter), ולא אוגר הזזה איטי (Shift Register) המבוסס על פליפ-פלופים. זה קריטי כדי שהסיבוב כולו יתרחש ב-0 זמן השהיה לוגי.

### שבלונה למימוש פעולת ARX (שרשור, XOR ו-Rotate):
כאשר מבקשים ממך לבצע הזזה מעגלית (Rotate), הדרך הנכונה ב-Verilog בר-סינתזה היא על ידי שרשור ביטים מחדש בעזרת סוגריים מסולסלים `{}`.

```verilog
`default_nettype none 

module ARX_Round (
    input  wire [31:0] x0, x1, // כניסות מידע (שים לב לאינדקסים בשרטוט!)
    input  wire [31:0] rk0, rk1, // מפתחות
    output wire [31:0] x_out // יציאה
);

    wire [31:0] a0; // חוט ביניים

    // שלב 1: חיבור מודולרי מלווה בפעולות XOR (Add & XOR)
    assign a0 = (x0 ^ rk0) + (x1 ^ rk1); 

    // שלב 2: הזזה מעגלית (Rotate Left / ROL)
    // דוגמה לביצוע הזזה של 9 ביטים שמאלה:
    // לוקחים את הביטים מ-9 ועד הסוף, ומדביקים להם את 9 הביטים הראשונים
    assign x_out[0:31] = {a0[9:31], a0[0:8]}; // אזהרה: אינדקסים הפוכים, בדוק במבחן!

endmodule
```

*(הערת מוקש: שים לב תמיד בשרטוט לאינדקסים - האם הווקטור מוגדר כ- `[31:0]` שזה הסטנדרט, או בצורה הפוכה `[0:31]`, וחווט את פעולת השרשור בהתאם!)*
