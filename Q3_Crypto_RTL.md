# 🧩 ענף 3: ארכיטקטורות קריפטוגרפיות ותכנון RTL (שאלה 3 במבחן)

**מהות השאלה:** יינתן שרטוט בלוקים של סיבוב בודד מתוך אלגוריתם הצפנה לא מוכר. יש לזהות את הארכיטקטורה שלו (Feistel / SPN / ARX) ולתרגם את השרטוט לקוד Verilog קומבינטורי נקי המבוסס על `assign`.

---

## 1. זיהוי ארכיטקטורות מתוך שרטוט
במבחן תישאל "לאיזו משפחה שייך האלגוריתם?", אלו האפשרויות:

* **Feistel Network:** * **איך מזהים?** המידע הנכנס מפוצל לשני חצאים (לרוב ימין ושמאל). חצי אחד עובר ללא שינוי, והחצי השני מפעיל פונקציה מורכבת (F) על החצי הראשון בעזרת מפתח. בסוף יש לרוב שילוב באמצעות XOR.
  * **דוגמאות במבחנים:** GOST, SEED.

* **SP Network (Substitution-Permutation):**
  * **איך מזהים?** כל בלוק המידע עובר במקביל פעולת החלפה (דרך קופסאות S-Box), ולאחר מכן ערבוב (Permutation) ללא פיצול הבלוק.

* **המוקש - רשת ARX (Add, Rotate, XOR):**
  * **איך מזהים?** אם השרטוט אינו מפצל מידע כמו ב-Feistel, ואין בו שום קופסאות S-BOX כמו ב-SPN, אלא רק חיבורים (+), XOR ופעולות הזזה מעגלית (<<<) – זהו מבנה ARX. 
  * **תשובה אופיינית למבחן:** "אין שום קשר או דמיון בין הארכיטקטורה ל-Feistel או SP Network".

---

## 2. המרת השרטוט ל-Verilog (Combinational RTL)
הקוד שתידרש לכתוב יהיה **תמיד** ללא בלוקים של שעון (`always @(posedge clk)`), אלא ישתמש בחוטים (`wire`) ובהשמות רציפות (`assign`) כדי לתאר התפשטות אותות.

### שבלונה למימוש פעולת ARX (שרשור, XOR ו-Rotate):
כאשר מבקשים ממך לבצע הזזה מעגלית (Rotate), הדרך הנכונה ב-Verilog בר-סינתזה היא על ידי שרשור ביטים מחדש בעזרת סוגריים מסולסלים `{}`. 

```verilog
`default_nettype none //

module ARX_Round (
    input  wire x0, x1, // כניסות מידע
    input  wire rk0, rk1, // מפתחות
    output wire x_out // יציאה
);

    wire a0; // חוט ביניים

    // שלב 1: חיבור מודולרי מלווה בפעולות XOR (Add & XOR)
    assign a0 = (x0 ^ rk0) + (x1 ^ rk1); //

    // שלב 2: הזזה מעגלית (Rotate Left / ROL)
    // דוגמה לביצוע הזזה של 9 ביטים שמאלה:
    // לוקחים את הביטים מ-9 ועד הסוף, ומדביקים להם את 9 הביטים הראשונים
    assign x_out[0:31] = {a0[9:31], a0[0:8]}; //

endmodule
```

*(הערת מוקש: שים לב תמיד בשרטוט לאינדקסים - האם הווקטור מוגדר כ- `[31:0]` שזה הסטנדרט, או בצורה הפוכה `[0:31]`, וחווט בהתאם!)*
