# 🔢 ענף 4: פולינומים, אוגרי הזזה ומנגנוני PRBS (שאלה 4 במבחן)

**מהות השאלה:** המרצה מספק פולינום מתמטי (למשל: $1+x^2+x^5$). עליך לתרגם אותו לייצוגים בינאריים, לקבוע את סוג ה-PRBS, לשרטט את מעגל החומרה (Fibonacci או Galois) ולכתוב לו קוד Verilog מבוסס פליפ-פלופים. בסוף נדרש למלא טבלת מצבי ביניים ו/או דיאגרמת מצבים.

---

## 1. המרות בסיסיות (המתמטיקה של PRBS)
* **זיהוי סוג ה-PRBS:** החזקה הגבוהה ביותר בפולינום ($L$) היא זו שקובעת את סוג ה-PRBS (את אורך השרשרת). לדוגמה: עבור $1+x^2+x^5$ החזקה הגבוהה היא 5, לכן המערכת נקראת **PRBS5**.
* **ייצוג בינארי והקסדצימלי:** רושמים "1" היכן שיש חזקה של $x$ (כולל "1" שזה למעשה $x^0$), ו-"0" היכן שהחזקה חסרה. 
  * *דוגמה:* הפולינום $1+x^2+x^3$ מיוצג בינארית כ-`1101` (קוראים מימין לשמאל: יש $x^0$, אין $x^1$, יש $x^2$, יש $x^3$), ובהקסדצימלי `h'd` (שזה 13 בעשרוני).
* **כמות מצבים ב-PRBS (אורך המחזור):** מחושבת תמיד לפי הנוסחה $2^L - 1$ (מחסירים 1 כי מצב שבו כל הפליפ-פלופים על 0 הוא מצב "תקוע" ואסור).
* **כמה '1' וכמה '0' יש בסדרה?** בסדרת PRBS תקינה הפילוג כמעט שווה, אך תמיד יש '1' אחד יותר:
  * **מספר האחדות ('1'):** $2^{L-1}$
  * **מספר האפסים ('0'):** $2^{L-1} - 1$

## 2. חוקי שרטוט וחומרה: Fibonacci מול Galois
לפני שניגשים לשרטט, נגדיר את רכיבי החומרה הדרושים (תקף לשתי השיטות):
* **מספר הפליפ-פלופים (FF):** תמיד שווה לחזקה הגבוהה ביותר בפולינום ($L$).
* **מספר שערי ה-XOR:** תמיד שווה ל-[כמות הפעמים שמופיע '1' בייצוג הבינארי] פחות 2. (לדוגמה: בפולינום $1+x^2+x^5$ הייצוג הוא `100101`. יש בו שלושה '1'. לכן נצטרך $3-2=1$ שערי XOR).

### א. ארכיטקטורת Fibonacci (פיבונאצ'י - "רבים ליחיד")
* **העיקרון:** שערי ה-XOR יושבים **מחוץ** לשרשרת הפליפ-פלופים. המידע נאסף מכמה נקודות, מחושב, ומוזן חזרה להתחלה.
* **איך לחבר פיזית?** * צייר שרשרת נקייה של פליפ-פלופים המחוברים זה לזה (מ-Q ל-D).
  * הסתכל על הפולינום: החזקות מראות מאיזה פליפ-פלופים למשוך חוט כלפי מעלה אל עץ ה-XOR.
  * *דוגמה ($1+x^3+x^4$):* משוך חוט מיציאת Q של FF3 וחוט מיציאת Q של FF4. חבר אותם לשער XOR. את התוצאה של ה-XOR חבר לכניסת ה-D של FF1.

### ב. ארכיטקטורת Galois (גלואה - "יחיד לרבים")
* **העיקרון:** שערי ה-XOR יושבים **בתוך** השרשרת (בין יציאה לכניסה). קו משוב אחד (Feedback) יוצא מהסוף ומזין את כולם.
* **איך לחבר פיזית?**
  * החזקה הגבוהה ביותר היא מקור הפידבק. משוך חוט ארוך מיציאת ה-Q של הפליפ-פלופ האחרון בשרשרת שחוזר אחורה לאורך כל השרטוט.
  * החזקות שבאמצע אומרות לך היכן לחתוך את החוט בין הפליפ-פלופים ולשים XOR. 
  * *דוגמה ($1+x^2+x^5$):* הפידבק יוצא מ-FF5. חתוך את החוט שנכנס ל-FF2 (בין FF3 ל-FF2) ושים שם XOR. ה-XOR הזה יקבל את המידע מ-FF3 ואת קו הפידבק מהסוף. בנוסף, חבר את קו הפידבק ישירות לכניסה של FF1.

## 3. שבלונות קוד Verilog (Synthesizable RTL)
הקוד חייב להיות סינכרוני, לכלול `reset`, ולהשתמש בהשמות לא-חוסמות (`<=`).
*שים לב: היצמד לאינדקסים בשאלה! אם המרצה כותב `reg [1:3]`, אל תשנה ל-`[2:0]`.*

### תבנית עבור Fibonacci (הזנה להתחלה והזזה רציפה)
כאן מעדכנים הכל בשורה אחת בעזרת שרשור. ביט ה-XOR מחושב ונכנס ראשון, ושאר הביטים זזים.
```verilog
module prbs ( 
    input  wire clk, 
    input  wire reset, 
    output wire prbs_out 
);
    reg [1:3] lfsr; // אוגר לפי גודל ה-PRBS

    always @(posedge clk or posedge reset) begin 
        if (reset)
            lfsr <= 3'h7; // ערך איתחול (יינתן בשאלה)
        else
            // הזזה: ה-XOR מוצב ראשון [1], והביטים 1-2 זזים ל-2-3
            lfsr <= { (lfsr[2] ^ lfsr[3]) , lfsr[1:2] }; 
    end
    assign prbs_out = lfsr[3]; // היציאה היא מהפליפ-פלופ האחרון
endmodule 
```

### תבנית עבור Galois (XOR פנימי)
ב-Galois לא משרשרים בשורה אחת בגלל ה-XOR-ים הפנימיים, אלא מבצעים השמה ספציפית לכל FF.
```verilog
module prbs (
    input clk, 
    input reset, 
    output prbs_out 
);
    reg [4:1] lfsr; 

    always @(posedge clk or posedge reset) begin 
        if (reset)
            lfsr <= 4'b1111; // איתחול
        else begin 
            // כל FF מקבל מהקודם, למעט אלו שיש לפניהם XOR מה-Feedback
            lfsr[1] <= lfsr[2] ^ lfsr[1]; 
            lfsr[2] <= lfsr[3]; 
            lfsr[3] <= lfsr[4]; 
            lfsr[4] <= lfsr[1]; // מקור ההזנה (Feedback)
        end
    end
    assign prbs_out = lfsr[1]; // היציאה הסופית
endmodule
```

## 4. מעקב מצבים ודיאגרמת מצבים (Desk-Check)

### א. איך מחשבים את הערכים הבאים בטבלה?
כדי לא לטעות, עבוד לאט ובשלבים על דף טיוטה:
1. **הצב את ערך האיתחול (Seed):** רשום מתחת לכל פליפ-פלופ בשרטוט את הערך ההתחלתי שלו (למשל `1 1 1`).
2. **חשב את ה-XOR:** לפני שאתה מזיז משהו, בדוק מה הערכים שנכנסים ל-XOR(ים) באותו רגע וחשב את התוצאה (0 או 1).
3. **בצע הזזה (עליית שעון):** עכשיו, הזז כל ביט צעד אחד קדימה. למקום הריק שנפתח (בפיבונאצ'י) או לתוך הפליפ-פלופ שאחרי ה-XOR (בגלואה) - הכנס את התוצאה שחישבת בשלב 2.
4. **חזור על הפעולה:** הערך החדש של הרגיסטר הוא השורה הבאה בטבלה. ביט היציאה (`prbs_out`) הוא הערך שנמצא בפליפ-פלופ האחרון בשרטוט במחזור הנוכחי.

### ב. איך לשרטט דיאגרמת מצבים (State Diagram)?
דיאגרמת מצבים היא הייצוג הגרפי של הטבלה שזה עתה חישבת:
* צייר עיגול ובתוכו רשום את המצב הנוכחי של הרגיסטר (למשל: `111`).
* מתח חץ מהעיגול הזה לעיגול חדש, ובתוכו רשום את המצב הבא שקיבלת לאחר מחזור השעון (למשל: `110`).
* המשך למתוח חצים בהתאם לחישובים שלך.
* מכיוון שמדובר בפולינום פרימיטיבי תקין, לאחר $2^L - 1$ צעדים החץ האחרון חייב לחזור לעיגול הראשון (`111`) ולסגור מעגל גדול. 
* *הערה:* המצב שבו כל הביטים הם '0' (למשל `000`) משורטט לרוב כעיגול נפרד בצד עם חץ שחוזר לעצמו, שכן זהו "מצב תקוע" שהמערכת לעולם לא תצא ממנו או תיכנס אליו באופן טבעי.
