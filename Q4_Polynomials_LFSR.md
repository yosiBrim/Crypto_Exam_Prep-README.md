# 🔢 ענף 4: פולינומים, אוגרי הזזה ומנגנוני PRBS (שאלה 4 במבחן)

**מהות השאלה:** המרצה מספק פולינום מתמטי (למשל: $1+x^2+x^5$). עליך לתרגם אותו לייצוגים בינאריים, לקבוע את סוג ה-PRBS, לשרטט את מעגל החומרה (Fibonacci או Galois) ולכתוב לו קוד Verilog מבוסס פליפ-פלופים. בסוף נדרש למלא טבלת מצבי ביניים.

---

## 1. המרות בסיסיות (המתמטיקה)
* **זיהוי סוג ה-PRBS:** החזקה הגבוהה ביותר בפולינום (L) היא זו שקובעת את סוג ה-PRBS. (לדוגמה: $1+x^2+x^5$ $\rightarrow$ החזקה הגבוהה היא 5, לכן זה PRBS5).
* **ייצוג בינארי והקסדצימלי:** רושמים "1" היכן שיש חזקה של $x$ (כולל "1" שזה למעשה $x^0$), ו-"0" היכן שהחזקה חסרה. 
  * *דוגמה מתוך מועד ב' תשפ"ד:* הפולינום $1+x^2+x^3$ מיוצג בינארית כ-`1101` (קוראים מימין לשמאל: יש $x^0$, אין $x^1$, יש $x^2$, יש $x^3$), ובהקסדצימלי `h'd` (שזה 13 בעשרוני).
* **כמות מצבים ב-PRBS:** מחושבת תמיד לפי הנוסחה $2^L - 1$.

---

## 2. חוקי שרטוט: Fibonacci מול Galois
במבחן תידרש לשרטט אחת משתי התצורות.
* **Fibonacci LFSR:** שערי ה-XOR נמצאים **מחוץ** לשרשרת הפליפ-פלופים. כל הפליפ-פלופים משורשרים אחד לשני בתור (Shift Register פשוט), וההזנה חזרה (Feedback) מחושבת למעלה ונכנסת רק לכניסת ה-D של הפליפ-פלופ הראשון (FF1).
* **Galois LFSR:** שערי ה-XOR נמצאים **בתוך** השרשרת (בין יציאה של FF אחד לכניסה של ה-FF הבא). ה-Feedback יוצא מהפליפ-פלופ האחרון ומזין במקביל את ה-D של FF1 ואת כל שערי ה-XOR שבאמצע.

---

## 3. שבלונות קוד Verilog (Synthesizable RTL)
הקוד חייב להיות סינכרוני (בתוך `always @(posedge clk)`), לכלול `reset` (רצוי א-סינכרוני לפי מבחני עבר), ולהשתמש בהשמות לא-חוסמות (`<=`).
*שים לב: במבחנים המרצה נוטה להשתמש באינדקסים וקטוריים הפוכים (למשל `reg [1:3]` במקום `reg [2:0]`). היצמד לאינדקסים בשאלה!*.

### תבנית עבור Fibonacci (מבוסס מועד ב' תשפ"ד)
כאן מעדכנים את הכל בשורה אחת בעזרת שרשור (Concatenation): ביט ה-XOR נכנס ראשון, ושאר הביטים זזים ימינה.

```verilog
module prbs ( //
    input  wire clk, //
    input  wire reset, //
    output wire prbs_out //
);

    reg [1:3] lfsr; // הגדרת אוגר בגודל ה-PRBS

    always @(posedge clk or posedge reset) begin //
        if (reset)
            lfsr <= 3'h7; // ערך איתחול (יינתן בשאלה)
        else
            // הזזה שמאלה: ה-XOR (Feedback) מוצב ראשון [1], והביטים 1-2 זזים ל-2-3
            lfsr <= { (lfsr[2] ^ lfsr[3]) , lfsr[1:2] }; //
    end

    assign prbs_out = lfsr[3]; // היציאה היא תמיד מהפליפ-פלופ האחרון

endmodule //
```

### תבנית עבור Galois (מבוסס מועד א' תשפ"ד)
ב-Galois לא ניתן לשרשר בשורה אחת בגלל ה-XOR-ים הפנימיים. לכן משתמשים בבלוק `begin...end` ומבצעים השמה ספציפית לכל פליפ-פלופ בנפרד.

```verilog
module prbs (
    input clk, //
    input reset, //
    output prbs_out //
);

    reg [4:1] lfsr; //

    always @(posedge clk or posedge reset) begin //
        if (reset)
            lfsr <= 4'b1111; // איתחול
        else begin //
            // כל FF מקבל את הערך מהקודם, למעט אלו שיש לפניהם XOR מה-Feedback
            lfsr[1] <= lfsr[2] ^ lfsr[1]; //
            lfsr[2] <= lfsr[3]; //
            lfsr[3] <= lfsr[4]; //
            lfsr[4] <= lfsr[1]; // ההזנה (Feedback)
        end
    end

    assign prbs_out = lfsr[1]; // היציאה הסופית מהמנגנון
endmodule
```

## 4. טבלת מעקב (Simulation Desk-Check)
בסוף השאלה תתבקש למלא 3 שורות ראשונות של טבלת מצבים לאחר האיתחול.
* **טיפ מעשי למבחן:** טעות קטנה בביט אחד בשורה הראשונה תהרוס את כל שאר השורות. שרטט לעצמך בצד דף המבחן את הפליפ-פלופים עם עיפרון, רשום בתוכם את מצב האיתחול (למשל `1 1 1`), מחק ותעדכן ידנית בכל "עליית שעון" לפני שאתה מעתיק את השורה לטבלה הסופית.
