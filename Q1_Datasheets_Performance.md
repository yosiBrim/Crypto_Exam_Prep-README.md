# 🌿 ענף 1: ממשקים, דפי נתונים (Datasheets) וניתוח ביצועים (שאלה 1 במבחן)

**מהות השאלה:** בשאלה זו יוצג לך דף נתונים (Datasheet) באנגלית של רכיב קריפטוגרפי מהתעשייה. המטרה היא כפולה:
1. לחלץ את רשימת הכניסות והיציאות ולכתוב עבורן "קליפת" מודול (Interface) ב-Verilog.
2. לדלות את נתוני הביצועים מתוך הטקסט או הטבלאות ולחשב מדדים הנדסיים כמו קצב העברת נתונים (Throughput) או זמן ריצה.


---

## שיטת ה"רדאר": 4 הנתונים לחילוץ (סדר פעולות למבחן)
אל תקרא את כל הטקסט! במבחן, רפרף על הדף וחפש אך ורק את 4 הנתונים הבאים לפי הסדר:

* **שלב 1 (למודול): שם הרכיב ורשימת ה-I/O.** חפש שרטוט קופסה עם חצים (Block Diagram) או טבלה בשם `Pin Description`. העתק את שמות הסיגנלים והרוחב שלהם (למשל `[31:0]`) אחד לאחד.
* **שלב 2 (לחישובים): תדר מקסימלי ($f_{max}$).** חפש טבלה בשם `Performance` או `Synthesis Results`. חפש את העמודה של `MHz` והוצא משם את התדר עבור הכרטיס המבוקש.
* **שלב 3 (לחישובים): גודל בלוק המידע (Block Size).** חפש בטקסט את המילים `block size` או `message`. (למשל: "512-bit message block"). זה יהיה המונה בנוסחה שלך.
* **שלב 4 (לחישובים): מחזורי שעון לעיבוד (Clock Cycles).** חפש בטקסט את המילה `cycles`. (למשל: "performed in 66 clock cycles"). זה המחנה בנוסחה.

## 1. כתיבת קליפת המודול (Verilog Interface)
**חוק ברזל:** חובה תמיד להתחיל את הקוד עם הנחיית הקומפיילר `default_nettype none` כדי למנוע יצירת רשתות סמויות במקרה של שגיאת כתיב. לאחר מכן, יש להגדיר את כל הכניסות (`input wire`) והיציאות (`output wire`) בדיוק לפי השרטוט וטבלת הפינים בדף הנתונים. אין צורך לכתוב לוגיקה פנימית, אלא רק את ה-I/O כדי לוודא שאתה יודע לעטוף את הרכיב לפני אינטגרציה.

**תבנית קוד אופיינית (מבוסס על פתרונות מהשנים האחרונות):**
```verilog
`default_nettype none // חובה בכל קוד!

module crypto_core_name ( // יש להחליף בשם הרכיב מהשאלה
    // כניסות שעון ואיפוס סטנדרטיות
    input  wire         clk, //
    input  wire         rst_n, //

    // כניסות בקרה ומידע
    input  wire         load, //
    input  wire [31:0]  din, // לשים לב היטב לרוחב הוקטור בטבלה
    
    // יציאות
    output wire         ready, //
    output wire [127:0] dout //
);
    // This answer only requires I/Os specifications
endmodule //
```

## 2. נוסחאות חומרה וביצועים (Performance Calculations)

במבחנים נדרש לבצע חישובים הנדסיים של זמנים וקצבים, תוך המרת יחידות בצורה מדויקת (Milli, Micro, Nano). אלו שתי הנוסחאות היחידות שתצטרך לעבוד איתן:

### א. חישוב זמן ביצוע (Execution Time)
כאשר ידוע מספר מחזורי השעון (Clock Cycles) הדרושים לפעולה, ותדר העבודה המקסימלי (Max Frequency).
* **הנוסחה:** $Execution\_Time = Cycles \cdot T = \frac{Cycles}{Frequency}$
* **דוגמה קלאסית (מתוך תשפ"ד מועד ב'):** רכיב דורש 180,000 מחזורי שעון בתדר של 212.27MHz.
  $Execution\_Time = 180,000 \cdot \frac{1}{212.27 \cdot 10^6} = 8.4797 \cdot 10^{-4} [sec] = 84.797 [ms]$.

### ב. חישוב קצב העברת נתונים (Throughput)
Throughput הוא כמות המידע (בביטים) המעובדת ביחידת זמן נתונה.
* **הנוסחה:** $Throughput = \frac{Block\_Size\_in\_Bits}{Execution\_Time\_per\_Block}$ או $Throughput = \frac{Block\_Size}{Cycles\_per\_Block \cdot T}$.
* **דוגמה קלאסית (מתוך תשפ"ד מועד א'):** רכיב בעל בלוק של 576 ביט, המסיים עיבוד ב-12 מחזורי שעון, כאשר זמן המחזור הוא 5.2938ns.
  $Throughput = \frac{576}{12 \cdot 5.2938 \cdot 10^{-9}} = 9.067 \cdot 10^9 [bps] = 9.067 [Gbps]$.
  
### 💡 קיצור דרך חישובי (The "Bits per Cycle" Ratio)
על פי פתרונות המרצה, מתמטית במקום לחלק בזמן המחזור (T), ניתן להכפיל בתדר (f). [cite_start]זה חוסך טעויות המרה (נאנו-שניות מול מגה-הרץ) כשמוצגת טבלה עם מספר כרטיסים בתדרים שונים[cite: 1000, 1001, 1002, 1010].
1. חשב את היחס הקבוע לאלגוריתם: `Ratio = Block_Size / Clock_Cycles`.
2. לכל כרטיס בטבלה, פשוט הכפל את היחס בתדר שלו: `Throughput = Ratio * Frequency`.
[cite_start]*(זהירות: אם מדובר באלגוריתם RSA כמו במבחן 20a, יש לחשב את ה-Clock Cycles ידנית בעזרת אלגוריתם Square and Multiply לפני שמבצעים את היחס [cite: 1458, 1459, 1466, 1467]).*

*(הערה חשובה: לפעמים השאלה הופכת את הנתונים – נותנים לך את ה-Throughput ומבקשים למצוא את מספר מחזורי השעון. במצב כזה, פשוט מציבים נעלם $N$ בנוסחה ומחלצים אותו אלגברית.)*
