# 🌿 ענף 1: ממשקים, דפי נתונים (Datasheets) וניתוח ביצועים (שאלה 1 במבחן)

**מהות השאלה:** בשאלה זו יוצג לך דף נתונים (Datasheet) באנגלית של רכיב קריפטוגרפי מהתעשייה (כמו SHA3, MD5, RSA). המטרה היא כפולה:
1. לחלץ את רשימת הכניסות והיציאות ולכתוב עבורן "קליפת" מודול (Interface) ב-Verilog.
2. לדלות את נתוני הביצועים מתוך הטקסט/טבלאות ולחשב מדדים כמו קצב העברת נתונים (Throughput) או זמן ריצה.

---

## 1. כתיבת קליפת המודול (Verilog Interface)
**חוק ברזל של אורי שטרו:** חובה תמיד להתחיל את הקוד עם הנחיית הקומפיילר `default_nettype none` כדי למנוע יצירת רשתות סמויות במקרה של שגיאת כתיב. לאחר מכן, יש להגדיר את כל הכניסות (`input wire`) והיציאות (`output wire`) בדיוק לפי השרטוט וטבלת הפינים בדף הנתונים. אין צורך לכתוב לוגיקה פנימית, רק את ה-I/O.

**תבנית קוד אופיינית (מבוסס על מבחני תשפ"ד):**
```verilog
`default_nettype none // חובה בכל קוד!

module crypto_core_name ( // להחליף בשם הרכיב
    // כניסות שעון ואיפוס סטנדרטיות
    input  wire         clk, //
    input  wire         rst_n, //

    // כניסות בקרה ומידע
    input  wire         load, //
    input  wire [31:0]  din, // שימו לב לרוחב הוקטור בטבלה
    
    // יציאות
    output wire         ready, //
    output wire [127:0] dout //
);
    // This answer only requires I/Os specifications
endmodule //
